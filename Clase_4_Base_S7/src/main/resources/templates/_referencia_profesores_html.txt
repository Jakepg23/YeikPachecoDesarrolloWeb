TAMBIEN: modificar templates/cursos/form.html para que use SELECT en vez de
texto plano para profesor.
---------------------------------------------------------------------------

En el form actual, busca el bloque del INPUT de "Profesor":

    <input type="text" class="form-control"
           th:classappend="${#fields.hasErrors('profesor')} ? 'is-invalid'"
           th:field="*{profesor}">

Reemplazalo por este SELECT (que recibe la lista de profesores del controller):

    <select class="form-select"
            th:classappend="${#fields.hasErrors('profesor')} ? 'is-invalid'"
            th:field="*{profesor}">
        <option value="">-- Selecciona un profesor --</option>
        <option th:each="p : ${profesores}"
                th:value="${p.id}"
                th:text="${p.nombre}"></option>
    </select>
    <div class="invalid-feedback" th:errors="*{profesor}">Error</div>

Conceptos:
  - th:field="*{profesor}" sobre un <select> genera name="profesor" en el form.
  - Cada option lleva el ID del profesor como value. Spring se encarga de
    convertir ese ID en una instancia de Profesor (cargandolo via Repository).
  - Esto funciona porque el campo "profesor" en Curso ahora es @ManyToOne Profesor.

---------------------------------------------------------------------------
Y en templates/cursos.html, donde se muestra "Profesor" en cada card,
cambiar:

    <span th:text="${c.profesor}">Profesor</span>

Por:

    <span th:text="${c.profesor?.nombre}">Profesor</span>

El operador ?. (safe navigation) evita NullPointerException si algun curso
viejo no tiene profesor asignado.
